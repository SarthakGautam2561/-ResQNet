# ResQNet (Disaster Response Platform)

This repo contains the mobile victim app and the responder dashboard for the ResQNet MVP.

## Repo Structure
- `D:\ResQNet\apps\mobile` - React Native (Expo) victim app
- `D:\ResQNet\apps\dashboard` - Vite + React responder dashboard
- `D:\ResQNet\supabase` - Database schema and policies
- `D:\ResQNet\packages\shared-types` - Shared type definitions

## Supabase Setup
1. Create a Supabase project.
2. In the Supabase SQL editor, run `D:\ResQNet\supabase\schema.sql`.
3. If your existing project used the old `acknowledged` status, run `D:\ResQNet\supabase\migrations\20260412_processed_status.sql`.
4. If your existing project lacks the `district` column, run `D:\ResQNet\supabase\migrations\20260412_add_district.sql`.
5. If you want to track multiple sources (friend + local), run `D:\ResQNet\supabase\migrations\20260412_add_source_project.sql`.
6. Optional: if you already have SOS rows and want processed + analytics immediately, run `D:\ResQNet\supabase\migrations\20260412_backfill_processed.sql`.
7. If you want friend‑processed data to always take priority, run `D:\ResQNet\supabase\migrations\20260412_prefer_friend_processed.sql`.
8. If you want **only friend processing** (no local processed rows until friend sync arrives), run `D:\ResQNet\supabase\migrations\20260412_disable_local_processing.sql`.
9. If your friend adds `detailed_area`, run `D:\ResQNet\supabase\migrations\20260412_add_detailed_area.sql`.
10. For performance indexes, run `D:\ResQNet\supabase\migrations\20260412_perf_indexes.sql`.
3. In Supabase Auth settings, confirm email confirmations are disabled (for hackathon speed) or use the admin script below to create seeded users.

## Environment Variables
No secrets are committed. Set environment variables locally.

### Mobile (Expo)
Create `D:\ResQNet\apps\mobile\.env.local`:
```
EXPO_PUBLIC_SUPABASE_URL=your_supabase_url_here
EXPO_PUBLIC_SUPABASE_ANON_KEY=your_supabase_anon_key_here
```
If you build with EAS, also add these as EAS environment variables. If you forget, the app now includes an in-app **Supabase Setup** screen to paste the URL/key once.

### Dashboard (Vite)
Create `D:\ResQNet\apps\dashboard\.env.local`:
```
VITE_SUPABASE_URL=your_supabase_url_here
VITE_SUPABASE_ANON_KEY=your_supabase_anon_key_here
```

### Admin Scripts (Optional)
For `D:\ResQNet\setup-db.js` or `D:\ResQNet\apps\dashboard\fix-trigger.js`, set:
```
SUPABASE_URL=your_supabase_url_here
SUPABASE_SERVICE_ROLE_KEY=your_supabase_service_role_key_here
```

## Notes
- The mobile app queues SOS reports offline and auto-syncs on reconnect.
- The dashboard subscribes to realtime `sos_reports` changes for live updates.
- Public view fetches a safe subset of SOS fields (no phone numbers).
- Processed intel (`sos_reports_processed`) and analytics (`disaster_analytics`) are auto-generated by database triggers.

## Friend DB Live Sync (Processed + Analytics)
To mirror your friend's processed data into your database, run (PowerShell):
```
$env:FRIEND_SUPABASE_URL="your_friend_supabase_url"
$env:FRIEND_SUPABASE_SERVICE_ROLE_KEY="your_friend_service_role_key"
$env:SUPABASE_URL="your_local_supabase_url"
$env:SUPABASE_SERVICE_ROLE_KEY="your_local_service_role_key"
node D:\ResQNet\scripts\sync-friend-data.js
```

Run this script repeatedly (cron/Task Scheduler) for near‑real‑time sync. Data is stored with `source_project = 'friend'` so it doesn't overwrite your local analytics.

### Edge Function (Optional Production Sync)
There is a Supabase Edge Function you can schedule with Supabase Cron:
`D:\ResQNet\supabase\functions\friend-sync\index.ts`

Set secrets in Supabase:
- `FRIEND_SUPABASE_URL`
- `FRIEND_SUPABASE_SERVICE_ROLE_KEY`
- `SUPABASE_URL`
- `SUPABASE_SERVICE_ROLE_KEY`

Then deploy the function and schedule it (e.g., every minute) to keep sync fully managed.

## Foreground Relay Mode (Android)
This build includes a foreground **Bluetooth Classic** relay (no Wi-Fi required).
- Offline device scans nearby Bluetooth devices and sends queued SOS to the relay phone.
- Online device runs Relay Mode and forwards received SOS to Supabase.

**To test:**
1. Install the dev client APK on both Android devices.
2. Start the dev server: `npx expo start --dev-client`
3. Open the app on both devices.
4. On the **online** device, open **Relay Mode** and tap **Start Relay Listener**.
5. On the **offline** device, open **Relay Mode**, tap **Scan for Relays**, select the relay device, then tap **Send Pending SOS**.
6. If pairing is required, tap **Bluetooth Settings** and pair once.
